THE CHALLENGE

The Problem We're Solving

Traditional banking apps in India often struggle to balance security with usability, resulting in cluttered interfaces, slow loading times, and a lack of clear financial visibility.

🔒

Security vs. Usability

Users want quick access but banks overload them with security steps. Finding the sweet spot between PIN, OTP, and biometric auth without friction.

💸

Complex Transactions

Fund transfers involve too many screens, unclear fees, and no confirmation that feels trustworthy.

📊

Financial Visibility

Most banking apps hide spending patterns behind reports no one reads. Users lack a quick pulse on their finances.

🕐

Speed & Responsiveness

Slow apps erode trust. A 3-second delay on a payment screen creates anxiety about whether the transaction went through.

🎨

Visual Overload

Traditional banking dashboards pack too many features into one screen, overwhelming first-time and regular users alike.

📱

Privacy in Public Spaces

Users check balances on metros, in offices, and coffee shops. Sensitive financial data needs smarter hide/show behavior.

THE MISSION

What We Aim to Achieve

Our primary goal is to redefine the mobile banking experience by focusing on clarity, speed, and privacy-first design principles.

01

Design a Secure Login Flow

Multi-step authentication combining email, OTP, and PIN with optional biometric login for returning users.

02

Create a Clean Dashboard

Surface pending bills, quick actions, and spending insights without overwhelming the user.

03

Simplify Fund Transfers

Custom keypad, real-time insufficient balance warnings, and a satisfying success animation with haptic feedback.

04

Build Financial Awareness

Interactive spending insights with pie charts breaking expenses into categories like Shopping, Food, Travel, and Bills.

05

Ensure Privacy by Default

Auto-hiding balances with a 7-second countdown timer that also triggers when the user switches apps.

06

Deliver a Premium Dark UI

Consistent dark theme across all screens with smooth, purposeful animations and micro-interactions.

THE JOURNEY

The Complete User Flow

Mapping out the end-to-end journey for a user navigating the Nova Bank application.

  • Splash Screen — 2-second branded intro with Nova Bank logo and loading spinner
  • Authentication — Email + password → 6-digit OTP verification → 4-digit PIN setup (new users)
  • Dashboard — Central hub: check balance, pay bills, view insights, access quick services
  • Transfer Money — Select account → Enter amount via custom keypad → Add recipient → Confirm → Success with haptic feedback
  • Transaction History — Filter by All, Income, or Expense with date-grouped entries
  • Card Management — View virtual debit card, toggle online/international transactions, copy card details
  • Profile & Settings — Edit personal info, manage security (PIN, biometric, 2FA), notification preferences

SECTION 4

Interaction Design

Transforming static screens into a fluid, context-driven experience that prioritizes user intent and security.

Seamless Navigation

The app structure is designed to feel instantaneous and cohesive, avoiding jarring page loads while preserving the user's context and momentum as they move between financial tasks.

Preventing Accidental Lockouts

Designed a mandatory 3-step PIN change architecture (Verify Old → Set New → Confirm New). This intentional friction prevents users from accidentally locking themselves out due to typos.

Contextual Authentication

Integrated a seamless Two-Factor Authentication flow. When high-security actions are detected, the standard flow is gently paused and the user is guided through an OTP verification sequence without losing their place.

Frictionless Input

Engineered smart auto-focusing for PIN and OTP fields. As a user types, focus shifts automatically to the next digit, and backspacing seamlessly returns to the previous input, making data entry completely fluid.

SECTION 5

Failure Paths & Exception Handling

Building a resilient financial app means planning for when things go wrong.

🚨

Consistent Error Handling

Built a single, unified alert system that surfaces clearly whenever something goes wrong — a failed network request, a declined transaction, or an authentication issue. It appears consistently over any screen so users always get the same clear, calm message instead of a silent failure or a confusing dead end.

🛡️

Real-Time Validation

Implemented immediate inline validation for critical interactions—such as ensuring 'Confirm PIN' perfectly matches 'New PIN' before persistence, flashing semantic red error states dynamically.

📡

Staying Informed Mid-Transaction

If a user's connection drops in the middle of a payment, the app immediately shows a dismissible alert rather than leaving them wondering whether the transaction went through. This was a deliberate choice for a banking app — silence during a financial transaction erodes trust faster than an honest error message.

Key Findings & Refinements

⏱️

Timer Perception

Participant 3 felt the 7-second balance countdown "felt like 4-5 seconds." The fast-paced animation creates urgency that may not match user expectations.

✓ IDENTIFIED & NOTED
⌨️

Keypad Auto-Focus

Participant 2 flagged that the amount field doesn't auto-trigger the keypad on the payment screen, requiring an extra scroll. A clear friction point.

✓ IMPLEMENTED IN APP
🔢

Balance Readability

Two participants misread ₹3,50,200 — one as ₹3,52,000 and another as ₹1,36,500. Currency formatting with proper thousand separators is critical.

✓ IMPLEMENTED IN APP
🔐

Public Space Privacy

Participant 4 raised concerns about viewing balances in crowded environments like metros. The auto-hide feature partially addresses this, but blur-on-look-away could enhance it further.

✓ IDENTIFIED & NOTED

"While paying, make sure the keypad gets popped up so that we don't have to scroll to put the amount."

— Participant 2, Usability Test

"7 seconds of balance screen feels like 4-5 seconds — can be increased by a few seconds."

— Participant 3, Usability Test

Tools & Resources

Figma + Figma Make

Utilized bleeding-edge AI design-to-code pipelines to scaffold initial React component structures from high-fidelity mockups. Directed the AI-executed implementation and validated output against the Figma design to ensure pixel-perfect fidelity and semantic architecture.

Google Fonts — Inter

Inter serves as the sole typeface across all screens — from 13px muted details to 39px display headings — ensuring consistency and exceptional readability.

Lucide Icons

A clean, consistent icon set integrated into the interactive prototype to provide crisp, scalable visual indicators.

Framer Motion

Orchestrated advanced fluid motion design and spring-physics configuration profiles via an AI-assisted development workflow. Directed the precise layout alignment, coordinate calculations, and exit/entry states to deliver a highly tactile, responsive mobile banking experience.
