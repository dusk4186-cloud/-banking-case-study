# Nova Bank — UX/UI & Engineering Case Study

This repository contains the interactive, web-based case study for the **Nova Bank Mobile Application** design and engineering project. 

It documents the journey from problem identification and high-fidelity Figma mockups to a fully functional React prototype, showcasing a deep integration of Design Systems and Frontend Engineering.

## 🌟 Case Study Highlights

This document serves as a living testament to the design aesthetic and technical constraints implemented throughout the project:

- **Context & Discovery**: Outlines the core problems of traditional fintech apps (clutter, slow validation anxiety) and defines our target user personas.
- **Systems Architecture**: Highlights our strict adherence to a **1.25 (Major Third) typography ratio** and WCAG 2.1 contrast compliance.
- **Interactive State Engineering**: Details the React state-driven routing, 3-step security gating for PIN resets, and the robust 2FA intercept flow.
- **Smart Input Mechanics**: Explains the programmatic auto-focus shifting and backspace logic engineered for seamless OTP and PIN entry.
- **Failure Paths**: Documents our resilient global `<ErrorBanner/>` infrastructure and real-time network state listening.
- **Usability Validation**: Includes real-world testing metrics and the iterative refinements applied post-feedback.

## 🛠️ Built With
- **HTML5 & Vanilla CSS**: For maximum layout control and structural harmony.
- **Figma & Figma Make**: For UI design and initial component scaffolding.
- **Framer Motion (in the React app)**: Orchestrating the spring-physics animations shown in the case study examples.
- **Puppeteer**: For automated, high-fidelity screenshot generation.
